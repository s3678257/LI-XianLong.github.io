Project ideas.

My project idea is creating a FAQ and implementing it as a chatter-bot. In 2016, someone stated that the transformation of the robot model will be more destructive and interesting than the transition from web to mobile applications over the past decade, indeed more and more people are willing to accept and use chatter-bot, such as the “Siri”in the iPhone or the family intelligent robot. In china, about 47% of people who work in media and 31% of marketers use chatter-bots and 52% of media people and marketers said that chatter-bots are primarily targeted as a source of information. In addition, the social platforms such as Facebook Messenger are the“habitat”of chatter-bots, 88% of media people and marketers have placed them on this platform.The report will show the description of the chatter-bot, and the Tools and Technologies or equipment needed to creating a chatter-bot. Also, it includes the the outcome and influences of this project if it success.

The chatter-bot which also called chatbot, is a computer program or an artificial intelligence which conducts a conversation via auditory or textual methods, technically, Chatter-Bot is a Python library that makes it easy to generate automated responses to a user’s input. Chatter-Bot uses a selection of machine learning algorithms to produce different types of responses so that it makes it easy for users to have conversations with chatter-bot. The chatter-bot can understanding and interacting conversationally with users, moreover, it also has specific natural language processing (NLP) capabilities to understand the context of a conversation in multiple languages. The NPL is a natural language processing technology, it is a technique for studying computer processing of human language, It includes syntactic and semantic analysis, information extraction, text mining and so on. Chatter-bot could help users do daily tasks. It could provide required information from the users and do useful actions such as search something to provide a continuous positive experience to the users. The order or information should be accessible through chatter-bot itself. Users do not need to go through other ways or apps for related additional information. The Chatter-bot can be able to provide all needful information within the messaging app making it a pleasant experience for the users, the operating mode could be similar to the Apple company’s “Hey, Siri!”. Also, the project will “trick”the user into thinking that they are communicating with a person but not a robot, the chatter-bot will be more humanize, such as change the response way to be like a human, like add “human pause” into the communications, to the utmost extend, customers believe that they are chatting with a human instead of a bot.

In order to achieve this project idea, the basic of this project is Python programming knowledge, and Slack API. It also need Pip and virtualenv tools to handle Python application dependencies. And a Slack account with access to the API for Python Slack client code base built by the Official Slack team and Slack API test token. Also need a persistent relational database or NoSQL backend such as PostgreSQL, MySQL or SQLite to save and retrieve user data.

When the project success, it would be a chatter-bot program, and users could have communication with it, and scan for keywords within the input, it with pull a reply with the most matching keywords, or the most similar wording pattern, from a database. If there have original problems, users could check online or ask professional people.
